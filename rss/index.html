Some very clever people advocate process isolation as a main point of Docker2, while others want an environment where "docker run" becomes a substitute for "start these 4 services and setup SSH+VNC for my development IDE"1. While I think both usage scenarios can contribute back to the Docker community at large, non-experts are hopelessly lost on how to properly setup a container-as-a-vm (or when that is/is not justified). Specifically, I wonder if we will see containers that exist solely to orchestrate other containers. Sure, there's a lot left to lock down (integrate advanced networking4, remove need for sudo, and many others), but I am excited to isolated an entire environment of containers inside a parent and control entire environment memory and CPU usage.
